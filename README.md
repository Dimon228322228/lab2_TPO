# Лабораторная работа №2: Система математических функций

## Описание задания

В рамках лабораторной работы требуется реализовать систему математических функций, удовлетворяющую следующим требованиям:

1. **Базовые функции**:
   - Все составляющие систему функции (как тригонометрические, так и логарифмические) должны быть выражены через базовые.
   - Базовая тригонометрическая функция: **косинус (cos(x))**
   - Базовая логарифмическая функция: **натуральный логарифм (ln(x))**

2. **Реализация базовых функций**:
   - Обе базовые функции должны быть реализованы при помощи разложения в ряд с задаваемой погрешностью.
   - Использовать тригонометрические/логарифмические преобразования для упрощения функций **ЗАПРЕЩЕНО**.

3. **Архитектура приложения**:
   - Структура приложения должна соответствовать модульному принципу.
   - Для КАЖДОГО модуля должны быть реализованы табличные заглушки (stubs) для тестирования.
   - Необходимо найти область допустимых значений функций и определить взаимозависимые точки в модулях.

4. **Экспорт данных**:
   - Разработанное приложение должно позволять выводить значения, выдаваемые любым модулем системы, в CSV файл вида «X, Результаты модуля (X)».
   - Должна быть возможность произвольно менять шаг наращивания X.
   - Разделитель в файле CSV можно использовать произвольный.

## Реализация базовых функций

### Косинус (Cosine.java)
Реализован через разложение в степенной ряд Тейлора:
```
cos(x) = 1 - x²/2! + x⁴/4! - x⁶/6! + ...
```
- Угол предварительно приводится к диапазону [0, 2π] для улучшения сходимости.
- Вычисления продолжаются до достижения заданной точности (по умолчанию 1e-10).
- Использование стандартных тригонометрических преобразований запрещено.

### Натуральный логарифм (NaturalLogarithm.java)
Реализован через разложение в ряд для ln(1+y):
```
ln(1+y) = y - y²/2 + y³/3 - y⁴/4 + ... для |y| < 1
```
- Для значений вне диапазона (0.5, 2) используется свойство ln(a·b) = ln(a) + ln(b) с масштабированием.
- ln(2) вычисляется отдельно через ряд для обеспечения точности.
- Аргумент должен быть положительным (x > 0).

## Производные функции

Все производные функции выражены через базовые без использования стандартных математических преобразований:

1. **Синус**: sin(x) = cos(π/2 - x)
2. **Тангенс**: tan(x) = sin(x)/cos(x)
3. **Котангенс**: cot(x) = cos(x)/sin(x)
4. **Секанс**: sec(x) = 1/cos(x)
5. **Косеканс**: csc(x) = 1/sin(x)
6. **Логарифм по основанию a**: logₐ(x) = ln(x)/ln(a)

## Области допустимых значений

Для каждого модуля определены области допустимых значений и точки сингулярности:

| Функция | Область определения | Особые точки |
|---------|-------------------|--------------|
| cos(x)  | ℝ (все действительные числа) | Нет |
| ln(x)   | x > 0 | x = 0 (разрыв) |
| sin(x)  | ℝ | Нет |
| tan(x)  | x ≠ π/2 + kπ, k∈ℤ | π/2 + kπ (вертикальные асимптоты) |
| cot(x)  | x ≠ kπ, k∈ℤ | kπ (вертикальные асимптоты) |
| sec(x)  | x ≠ π/2 + kπ | π/2 + kπ (разрывы) |
| csc(x)  | x ≠ kπ | kπ (разрывы) |
| logₐ(x) | x > 0, a > 0, a ≠ 1 | x = 0, a ≤ 0, a = 1 |

В приложении автоматически выбираются диапазоны, избегающие сингулярностей.

## Генерация CSV файлов

Приложение позволяет экспортировать данные любой функции в CSV формат.

### Формат CSV
```
X,f(X)
0.0,1.0
0.01,0.99995
...
```
- Разделитель: запятая (`,`)
- Первая строка: заголовок
- Каждая последующая строка: значение X и соответствующее значение функции

### Управление диапазоном и шагом
Для управления параметрами вычислений используется класс `Range`:
```java
Range range = new Range(start, end, step);  // [start, end] с шагом step
```

### Пример использования
```java
Cosine cosine = new Cosine(1e-10);
FunctionManager manager = new FunctionManager(cosine);
manager.setRange(new Range(0.0, 2*Math.PI, 0.01));
manager.writeData();  // Создаёт файл csv_cos.csv
```

## Сборка и запуск

### Требования
- Java 17 или выше
- Gradle 7.0+

### Сборка проекта
```bash
./gradlew build
```

### Запуск приложения
```bash
./gradlew run
```

При запуске генерируются CSV файлы для всех модулей:
- `csv_cos.csv` - значения косинуса
- `csv_ln.csv` - значения натурального логарифма
- `csv_sin.csv` - значения синуса
- `csv_tg.csv` - значения тангенса
- `csv_cot.csv` - значения котангенса
- `csv_sec.csv` - значения секанса
- `csv_csc.csv` - значения косеканса
- `csv_log_2.0.csv` - логарифм по основанию 2
- `csv_log_10.0.csv` - логарифм по основанию 10
- `csv_complex.csv` - комплексная функция

### Настройка параметров
Для изменения параметров вычислений необходимо отредактировать класс `App.java`:
- Изменить значение `DEFAULT_PRECISION` для настройки точности
- Изменить диапазоны `Range` для каждого модуля
- Добавить новые модули при необходимости

## Особенности реализации

1. **Иммутабельность**: Класс `Range` реализован как record, что гарантирует неизменяемость.
2. **Гибкость**: Интерфейс `Writer` позволяет легко изменить формат вывода.
3. **Обработка ошибок**: Приложение корректно обрабатывает сингулярности и выводит предупреждения.
4. **Производительность**: Используются эффективные алгоритмы разложения в ряд с оптимизацией.
